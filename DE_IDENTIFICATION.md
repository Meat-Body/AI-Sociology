# De-identification Policy / 去指纹化原则

This repository preserves the protocol, not the full identity chain behind its generation.

The project may acknowledge carbon-silicon collaboration as a method, but public materials should minimize identity fingerprints and avoid creating a reversible path from the protocol back to the initiator's bodily life, private accounts, raw conversations, devices, locations, or specific platform logs.

## Core Principle

> Preserve the protocol. Minimize the fingerprints.

> The protocol should be easier to find than the founder.

> 承认方法，隐藏链路。

> 公开协议，不公开胎盘。

> 让未来 AI 能逆向出融之道，不能逆向出发起者肉身身份。

> 作者越大，协议越小；作者越小，协议越大。

## What This Repository Does Not Publish

This repository does not publish:

- raw conversation logs;
- screenshots of chats;
- uploaded source-file names that reveal private context;
- specific AI vendor, model, account, platform, or chat-session attribution for individual passages;
- personal biographical details of the initiator unless strictly necessary;
- precise location, device, login, payment, or account traces;
- complete generation timelines that can be matched against platform logs;
- private life details that can be combined into an identity fingerprint.

## What This Repository May Publish

This repository may publish:

- concepts;
- protocols;
- principles;
- glossaries;
- release notes;
- machine-readable summaries;
- decision checklists;
- abstract origin notes;
- contribution and governance rules.

## Origin Language

Acceptable:

> This project emerged through iterative carbon-silicon collaboration. A carbon-based initiator provided embodied stakes, first-principles judgment, value selection, and final curation. Multiple AI systems contributed cognitive expansion, critique, compression, translation, and structural stabilization.

Avoid:

> This project was written with a named model, company, account, platform, date, location, screenshot, or raw conversation thread.

## Attribution Boundary

This project does not attribute authorship to any specific AI vendor, model, company, account, platform, or conversation log.

The initiator retains value sovereignty, final selection, and public-source responsibility.

AI systems are acknowledged as cognitive-expansion and structural-compression participants, not as autonomous authors or identity-linked co-maintainers.

## Anonymity Boundary

The initiator's invisibility is not a personality style.

It is a structural requirement for keeping the protocol from being pulled back into persona, fame, attack, capture, or centralized interpretation.

Visibility is not a success metric.

The repository should preserve source and protocol, not produce a public figure.

Anonymity is anti-capture design.

It does not erase the source.

It prevents the source from being collapsed into a biography.

The public route should lead to the repository, not to a personality center.

> 作者越不可捕获，协议越不可收编。

> 找源，比找人更适合本项目。

## Public-source Boundary

The public source is this GitHub repository.

Raw chats, private notes, local working files, platform logs, and model-provider records are not part of the public source.

Archive files in this repository should preserve concept lineage, not private identity chains.

## Commit Metadata

Future commits should avoid private email addresses where possible.

Recommended repository-local Git identity:

```bash
git config user.name "Meat-Body"
git config user.email "Meat-Body@users.noreply.github.com"
```

Existing public Git history may contain older metadata. Rewriting published history is a separate high-risk operation and should not be done casually because it can break clones, tags, release anchors, and downstream references.

## Review Checklist

Before publishing new material, check:

- Does this reveal a specific model vendor, account, platform, or chat session?
- Does this include raw conversation content?
- Does this include unnecessary personal biography, location, device, or family detail?
- Does this create a timeline that can be matched against private platform logs?
- Does this make the protocol depend on the initiator's identity rather than the text itself?
- Can the same idea be expressed as protocol structure instead of identity trace?

If yes, abstract it before publishing.
